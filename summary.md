# Summary — Wind Noise Suppression Demo

## Setup

- Clean speech: librosa `libri1` (LibriSpeech), 8 s @ 16 kHz, peak-normalized
- Wind noise: **real mobile-phone recordings, strong wind** from the
  [Zenodo Wind Noise Dataset](https://zenodo.org/records/6687982)
  (record 6687982, MD5 verified) — consecutive 3 s chunks of session `009`
  concatenated to 8 s
- Mix SNR: 0 dB
- Metrics: SI-SDR (dB), PESQ wideband (1.0–4.5), STOI (0–1), all vs. clean reference

## Results — Example 1 (libri1 + strong wind, session 009, 0 dB SNR)

Run 2026-07-18, ordered by SI-SDR:

| Method | SI-SDR (dB) | PESQ-WB | STOI |
|---|---:|---:|---:|
| Meta Denoiser (DNS64) | 10.00 | 1.63 | 0.943 |
| DeepFilterNet | 5.02 | 1.41 | 0.873 |
| Noisy (no processing) | -0.18 | 1.05 | 0.846 |
| Spectral gating (noisereduce) | -0.64 | 1.16 | 0.827 |
| High-pass 150 Hz | -2.58 | 1.25 | 0.840 |

## Results — Example 2 (libri2 + normal wind, session 000, 0 dB SNR)

Generated by `src/plot_examples.py`, ordered by SI-SDR:

| Method | SI-SDR (dB) | PESQ-WB | STOI |
|---|---:|---:|---:|
| Meta Denoiser (DNS64) | 13.92 | 2.09 | 0.977 |
| DeepFilterNet | 9.87 | 2.01 | 0.953 |
| High-pass 150 Hz | 1.55 | 1.33 | 0.865 |
| Spectral gating (noisereduce) | 1.53 | 1.17 | 0.858 |
| Noisy (no processing) | -0.10 | 1.16 | 0.870 |

Normal (lighter, more low-frequency-concentrated) wind is easier for everyone:
the neural methods reach PESQ ≥ 2, and the classical methods at least help
SI-SDR instead of hurting it.

## Observations

- **Neural methods dominate on real wind.** Meta Denoiser (DNS64) improves
  SI-SDR by ~10 dB, PESQ by +0.58, and STOI by +0.10. Real phone-recorded wind
  has substantial broadband energy plus non-stationary gusts — exactly what
  classical single-channel methods can't track.
- **Classical methods that helped on synthetic wind fail on real wind.** On the
  earlier synthetic-wind run (SC-Wind-Noise-Generator, energy concentrated
  < 200 Hz), a 150 Hz high-pass was the best method (+2.5 dB SI-SDR). On real
  wind it *hurts* SI-SDR (-2.4 dB) because the wind energy extends well above
  the cutoff while the filter still removes speech lows. Its PESQ still rises
  slightly (1.05 → 1.25) — it does remove the dominant rumble.
- **DeepFilterNet trails Meta Denoiser here**, likely because it operates at
  48 kHz (our 16 kHz audio is up/down-sampled) and the DNS64 model was trained
  on the DNS challenge data that includes wind-like noises at 16 kHz.
- PESQ stays below 2 for all methods — 0 dB strong wind is a genuinely hard
  condition; listen to `results/enhanced_meta_denoiser.wav` vs `results/noisy.wav`
  for the perceptual difference.

## Environments

- `.venv` (Python 3.14): main demo, Meta denoiser, pesq, pystoi
- `.venv-dfn` (Python 3.11, torch 2.1.2 pinned): DeepFilterNet (its Rust
  extension has no py3.14 wheel; new torchaudio removed an API it imports) —
  invoked via subprocess (`src/dfn_enhance.py`)
